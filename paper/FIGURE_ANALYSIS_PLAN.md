# Figure And Analysis Plan

Temporary planning file for graphs to build after the full result set is merged.

## Existing Figures

Generated by `results/util/make_paper_figures.py` into `paper/figures/`.

- `model_calibrated_pass_rate`: headline raw/calibrated pass rate and compile success by model.
- `task_outcome_matrix`: model-by-task outcome grid.
- `time_vs_calibrated_pass_rate`: wall-clock time versus calibrated pass rate.
- `solved_by_model_count`: number of models that solve each task.
- `requirement_category_distribution`: benchmark requirement-category mix.
- `pass_rate_by_requirement_category`: pass rate by requirement category.
- `pass_rate_by_tier`: pass rate by task tier.
- `failure_mode_distribution`: aggregate failure mode distribution.
- `figure_summary.csv`: model/run summary table.
- `task_outcome_matrix.csv`: task outcome source table.

## Core Paper Figures To Add Or Improve

1. **Unreal subsystem pass rate**
   - Group tasks by subsystem: GAS, replication/networking, UI/HUD, animation, physics, save/load, editor/K2, asset/data systems, AI.
   - Show model pass rate per subsystem.
   - Purpose: make the Unreal-specific difficulty visible.

2. **Validation funnel**
   - For each model: attempted -> compiled -> static checks passed -> runtime/editor tests passed -> calibrated pass.
   - Purpose: separate code-writing ability from Unreal build/runtime correctness.

3. **Complexity versus success**
   - Scatter or binned plot of success against task complexity.
   - Candidate x-axes: files touched in reference solution, reference diff LOC, number of requirements, number of validators, or task prompt length.
   - Purpose: show how performance degrades with long-horizon implementation complexity.

4. **Requirement-category heatmap by model**
   - Model x requirement category, with pass rate or requirement satisfaction rate.
   - Purpose: reveal capability gaps hidden by task-level pass/fail.

5. **Failure mode by subsystem**
   - Stacked bars: subsystem x failure mode.
   - Purpose: distinguish compile/API failures from lifecycle, replication, asset/editor, or behavioral failures.

6. **Unique solves and overlap**
   - Per-task solved-by count plus model-unique solves.
   - Could be a compact upset-style table or heatmap rather than a full UpSet plot.
   - Purpose: show complementarity between agents/models.

## Secondary / Appendix Figures

1. **Cost-normalized performance**
   - Calibrated pass rate versus cost.
   - Use only runs with reliable cost accounting.

2. **Time-normalized performance**
   - Calibrated pass rate versus wall-clock time and per-task median time.
   - Separate pending/partial runs from complete runs.

3. **Patch size distribution**
   - Reference solution size and model solution size, if model patches are available.
   - Could include files changed, added LOC, deleted LOC.

4. **Task metadata distribution**
   - Tier, subsystem, requirement category, validator type, and task source/project.
   - Purpose: document benchmark composition.

5. **Validator type pass rate**
   - Static, editor inspection, config/scene, runtime behavioral, pipeline/frame.
   - Purpose: show which validation surfaces are hardest.

6. **Per-task difficulty table**
   - Task id, title, subsystem, tier, solved-by count, dominant failure mode.
   - Likely appendix/table rather than main figure.

7. **Qualitative failure examples**
   - 3-5 short case studies with task, expected behavior, model failure, and why Unreal-specific reasoning mattered.

## Data Needed

- Keep `paper/figures/figure_summary.csv` and `task_outcome_matrix.csv` as machine-readable outputs for tables.
